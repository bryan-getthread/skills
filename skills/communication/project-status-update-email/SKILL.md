# Project Status Update Email

Draft a client-facing status update for an ongoing project: what's moved since the last update, what's coming next, and anything the client needs to decide or unblock. Keeps stakeholders confident without a meeting.

## When to use

- "Send <client> the weekly update on the migration project."
- "Draft a project status email for this ticket."
- A project ticket needs a periodic update to the client sponsor.

## Steps

1. Read the project ticket, recent notes, and the prior update before writing so the new update is a true delta, not a repeat. Base progress claims only on recorded work.
2. Lead with the headline: on track / ahead / at risk, in one honest line.
3. Cover, in short sections:
   - **Done since last update** — completed milestones/tasks, plain language.
   - **In progress / next** — what's underway and what's next, with dates only where they're actually set.
   - **Needs you** — any decision, approval, or access the client must provide, made unmissable.
   - **Blockers / risks** — anything threatening timeline or scope, stated factually with the plan to address it.
4. If the project is at risk, say so plainly and pair it with the mitigation — never hide a slip behind optimistic language.
5. Structure and tone per the Email Baseline Standard (communication/email-baseline-standard).
6. Present the draft with view_openDraft for review. Do not send.

## Guardrails

- **Honest status only.** Report progress and dates as recorded, not as hoped. "We'll confirm the cutover date by <day>" beats a guessed date. Never convert a plan into a claimed completion.
- **No scope commitments.** Don't agree to added scope, new dates, or extra work in a status email — route those to the project owner/AM.
- Keep it a status update, not a change request; flag scope questions rather than resolving them here.
- Degradation: view_openDraft is in-app only. Over external MCP, output the draft in chat labeled "DRAFT — review before sending."
- For close-out use communication/resolution-closing-email; for a recurring account digest see account-management/vcio-weekly-proactive.
